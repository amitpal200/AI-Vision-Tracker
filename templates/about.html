About the Project

This system captures frames with OpenCV, runs YOLOv8 object detection, passes detections into Deep SORT, overlays labels and IDs, and stores every event in SQLite for dashboards and exports.

Pipeline

  1. OpenCV reads a webcam frame or uploaded video frame.
  2. YOLOv8 predicts COCO class boxes and confidence scores.
  3. Deep SORT associates detections across frames using motion and appearance.
  4. Annotated frames stream to the browser or are written to processed video.
  5. SQLite stores timestamped class names, confidence, IDs, boxes, and filenames.

Error Handling

Missing model weights, camera access failures, unsupported uploads, unreadable videos, and low-memory conditions are returned as clear UI/API errors.

The app selects CUDA automatically when PyTorch detects a compatible GPU, otherwise it uses CPU.
